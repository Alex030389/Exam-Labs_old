диск d/web/new-template - работаем с этим 

Стек технологий встроенных в сборщик gulp:

1) sass, pug, babel, sprite svg, sprite png
2) чтобы запустить сборку npm start (dev mode)
3) npm run build - это prod mode, который собирает prod версию и минифицирует css/js
4) в папке source находятся все исходники (img, pug, sass, svg, png images, js)
5) в папке template - находится все шаблоны --> папка common - это общие шаблоны, которые инклудяться на страницы --> папка pages - страницы, _template - это общий шаблон который инклудиться на страницах
6) папка style - стили css --> в папке pages много цсс файлов, не обращай внимание, а сразу открывай index.scss и пиши код туда
7) папка sprite - это sprite.svg - кладешь туда свг картинку - в консоли выполняешь gulp sprite:svg и у тебя билдится новый спрайт в папку build 
8) папка js -  в ней есть файл app.js в который инклудяться все остальные файлы
9) папка images - большие картинки
10) папка icons - иконки, которые командой в консоли gulp sprite:png собирают png спрайт
11) папка fonts - фонты
12) различные scss миксины (круг, квадрат и тд..) лежат в Style -> config -> mixins (variables)
